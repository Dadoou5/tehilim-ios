## 1. Chaîne de Tehilim (feature collaborative)

Lecture collective des 150 Tehilim répartie entre des participants qui ne se
connaissent pas forcément (partage par lien / QR), pour une **intention**
(Lelouy Nichmat · Refoua Chelema · Réussite).

C'est la **seule** partie de l'app qui sort de l'appareil. Sans configuration
backend dans la build, elle est **désactivée** et l'app reste 100 % locale.

### Cycle de vie d'une chaîne
1. **Création** (maître) : intention + détail, **durée de sélection**, **échéance
   de lecture**. Partage **lien WhatsApp** (`/c/?id=…` + scheme `tehilim://chain`)
   et **QR code** (modules bleu nuit + logo de l'app au centre).
2. **Sélection** (temps réel) : chacun rejoint avec un prénom (bascule optimiste
   immédiate), choisit ses Tehilim. Un Tehilim pris est **verrouillé** — verrou
   exclusif garanti par la **PK `(chain_id, psalm_id)`** (pas de transaction
   applicative). Désélection possible tant que non distribué.
3. **Clôture / distribution** (maître) : « m'attribuer les restants », **prolonger
   la sélection** (par durée ≤ 48 h, réarme les rappels), **clôturer & distribuer**.
4. **Lecture** : chacun lit ses Tehilim jusqu'à l'échéance ; compte rendu
   partageable.
5. **Fin** : suppression serveur automatique à *fin de lecture + 7 jours*.

### Liste « Mes chaînes » (3 catégories)
| Catégorie | Condition | Compte à rebours |
|---|---|---|
| **Sélection en cours** | non distribuée & `now < selectionDeadline` | → fin de sélection |
| **Lecture en cours** | distribuée / sélection close & `now < readingDeadline` | → fin de lecture |
| **Terminées** | `now ≥ readingDeadline` | — |

Compte à rebours `HH:MM:SS` (gros + gras), **bascule automatique** (recalcul
continu), **suppression locale** (Lecture/Terminées). Ouverture intelligente :
Sélection → détail live ; Lecture/Terminée → **lecteur hors-ligne** si une archive
locale existe.

### Notifications push (participants)
Via l'Edge Function `notify` (APNs + FCM, envois parallélisés) :
seuils **70/80/90/100 %**, **distribution**, **suppression**, **prolongation**,
rappels temporels **« il reste N »** (80 % du temps) et **« dernière chance »**
(95 %). Déclencheurs : triggers Postgres (pg_net) + `pg_cron` (toutes les 5 min).

### Lecture hors-ligne
À la distribution, un **instantané local** est conservé pour chaque participant
(`ChainArchiveStore` / `ChainArchiveSnapshot`, incluant *mes* Tehilim). Depuis
« Mes chaînes » → on ouvre un **lecteur hors-ligne** (grille de mes Tehilim →
texte 100 % embarqué) : fonctionne en **mode avion**.

## 2. Backend Supabase

- **Postgres** : `chains`, `chain_participants`, `chain_assignments`
  (PK `(chain_id, psalm_id)`), `device_tokens`. **Row Level Security** active —
  un utilisateur n'écrit que ses données ; le créateur a des droits étendus.
- **Auth anonyme** (identifiant aléatoire par appareil ; `awaitInitialization`
  côté Android pour ne pas recréer d'uid au retour d'arrière-plan).
- **Realtime** : `postgresChange` → refetch des collections.
- **RPC** : `create_chain`, `assign_remaining`, `remove_participant`,
  `extend_chain_selection`, `cleanup_expired_chains`, `process_chain_reminders`.
- **Edge Function `notify`** (Deno, `verify_jwt=false` + secret partagé) :
  APNs (JWT ES256) + FCM v1 (service-account), fallback sandbox⇄prod, purge des
  jetons morts (410 / UNREGISTERED).
- **pg_cron** : `chain_selection_reminders` (`*/5 * * * *`),
  `chain_cleanup_expired` (quotidien). Cron GitHub Actions = keep-alive
  (anti-pause 7 j) + appel de secours du cleanup.
- Migrations : `supabase/migrations/` · fonction : `supabase/functions/notify/`.
- Secrets (jamais commités) : `NOTIFY_SHARED_SECRET`, `APNS_*`, `FCM_SERVICE_ACCOUNT`,
  `FCM_PROJECT_ID`. Cf. `push-setup.md`.

**Config client (hors dépôt, gitignorée) :**
- iOS : `ios/Tehilim/Resources/Supabase-Info.plist`.
- Android : `app/supabase.properties` (→ BuildConfig) + `app/google-services.json` (FCM).

## 3. Lecture — réglage de la taille du texte

- Bouton **« Aa »** dans la barre de lecture → A− / A+ (aperçu live), **persisté**
  (mêmes préférences que Réglages → synchronisé). Ajuste le texte principal
  (hébreu/phonétique) et, si affichée, la traduction.
- Disponible sur : **détail d'un Tehilim**, **sections du Tehilim 119**, **lecture
  depuis une chaîne**.
- **8 paliers** : Très petit · Petit · Moyen · Grand · Très grand · Énorme · Géant ·
  Maximum (iOS scale 0.85→2.30 ; Android 0.85→2.00).
- **Numéro du Tehilim** affiché dans le contenu (toujours visible). Compte à rebours
  de lecture fiable via `TimelineView` (iOS).

Implémentation partagée : `ReadingTextSizeControl` (iOS), `ReadingTextSizeButton`
(Android) ; `TextSize.stepped()/isSmallest/isLargest`.

## 5. Confidentialité (résumé)

Lecture = aucune collecte, hors-ligne. Chaîne = uniquement les données saisies
(prénom d'affichage, intention, répartition) + un jeton push, le temps de la
chaîne, supprimées à *fin de lecture + 7 jours*. Détails : `PRIVACY.md`.
