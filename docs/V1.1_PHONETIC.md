# V1.1 — Phonétique sépharade

> Document consolidé. Contributions par agent.

---

## 1. Chef de projet — Cadrage

### Vision V1.1
Permettre à un lecteur francophone non hébraïsant de réciter les Tehilim grâce à une transcription latine (sépharade), sans imposer ce mode aux lecteurs hébraïsants.

### Périmètre
- Mode "Phonétique" sélectionnable dans Réglages → **Texte du psaume → Mode du texte**.
- Hébreu = mode par défaut, garanti.
- Le mode phonétique **remplace** l'hébreu (n'apparaît pas en plus). La traduction française reste pilotée par sa propre option.
- Système : sépharade uniquement en V1.1. Achkénaze candidat V2.

### Hors périmètre V1.1
- Transcription achkénaze.
- Transcription académique (ISO 259, SBL).
- Audio (la phonétique reste textuelle).
- Corpus pré-saisi par un humain (V2 si besoin de qualité supérieure).

### Critères d'acceptation
- [x] Picker "Mode du texte" dans Réglages avec valeurs `Hébreu` et `Phonétique`.
- [x] Au switch, tous les Tehilim affichés bascule en mode phonétique.
- [x] Préférence persiste après relance.
- [x] Footer informatif sous le picker en mode phonétique : "transcription assistée".
- [x] Tetragramme יהוה rendu par "Adonaï" (jamais transcrit littéralement).
- [x] Build vert + suite XCTest verte.

### Risques
- **Erreurs phonétiques** : la transcription est algorithmique, pas humaine. Étiquetée "assistée".
- **Sensibilité religieuse** : le tétragramme est protégé. Les autres noms divins suivent leur transcription naturelle.
- **Confusion utilisateur** : un utilisateur pourrait croire que la phonétique est validée. Footer informatif explicite.

## 2. Expert Tehilim — Règles de transcription

### Système retenu : sépharade

Choix justifié par la cible francophone (majoritairement sépharade en France, et plus accessible aux apprenants).

### Tableau des correspondances

#### Consonnes
| Lettre | Nu | + dagesh | Final |
|--------|-----|----------|-------|
| א | (muet) | – | – |
| ב | v | b | – |
| ג | g | g | – |
| ד | d | d | – |
| ה | h | h (mappiq) | (souvent muet, *non détecté V1.1*) |
| ו | v ou voyelle | ou (shuruq) | – |
| ז | z | z | – |
| ח | h | – | – |
| ט | t | t | – |
| י | y ou voyelle | y | – |
| כ / ך | kh | k | kh |
| ל | l | l | – |
| מ / ם | m | m | m |
| נ / ן | n | n | n |
| ס | s | s | – |
| ע | (muet) | – | – |
| פ / ף | f | p | f |
| צ / ץ | ts | ts | ts |
| ק | k | k | – |
| ר | r | r | – |
| ש (avec point droit) | ch | ch | – |
| ש (avec point gauche) | s | s | – |
| ת | t | t | – |

#### Voyelles (nikud)
| Marque | Phonétique |
|--------|-----------|
| ְ shva | (silencieux par défaut) |
| ֱ hataf segol | é |
| ֲ hataf patah | a |
| ֳ hataf qamats | o |
| ִ hiriq | i |
| ֵ tsere | é |
| ֶ segol | é |
| ַ patah | a |
| ָ qamats | a |
| ֹ holam | o |
| ֻ qubuts | ou |
| ׇ qamats qatan | o |

### Cas spéciaux (implémentés)

- **Tetragramme יהוה** → `Adonaï` (avec ou sans nikud).
- **וֹ (vav + holam)** → `o` (lettre traitée comme voyelle).
- **וּ (vav + dagesh seul)** → `ou` (shuruq).
- **ḥiriq + yod nu** → `i` (yod muet, mater lectionis).
- **Maqqef ־** → tiret `-`.
- **Sof pasuq ׃** → espace (séparateur de verset).

### Cas spéciaux non gérés (V1.1) — limites documentées

- **He final muet sans mappiq** : תּוֹרָה rendu `tora**h**` au lieu de `tora`. Acceptable phoniquement.
- **Qamats qatan contextuel** (kol = כָּל) : rendu `kal` au lieu de `kol`.
- **Shva na' / shva nah** : tous traités comme silencieux (shva nah). Le shva na' donne un "e" en vraie lecture (ex: שְׁמַע = "shema" et non "shma" parfois).
- **Begadkefat post-vocalique** : pas de mutation contextuelle.
- **Ketiv / Qere** : non géré (texte tel qu'écrit).

## 3. UX — Parcours utilisateur

### Flow principal

```
[Réglages]
   │ tap "Mode du texte"
   ▼
[Picker → Phonétique]
   │ choix
   ▼ (immédiat, sans confirmation)
[Tous les écrans Tehilim affichent désormais la phonétique]
```

### Réversibilité
Un retour sur Hébreu via le même picker rétablit l'affichage originel. Aucune perte de données.

### Découvrabilité
- Le picker se trouve en première position de la section "Texte du psaume" pour qu'il soit vu avant les pickers de taille.
- Footer explicite sous le picker quand `Phonétique` est sélectionné.
- Aperçu live dans la section.

### Pas d'onboarding spécifique
La feature est accessible mais discrète. Un utilisateur hébraïsant peut ne jamais la découvrir, ce qui est l'intention.

## 5. Designer — Direction visuelle

### Typographie phonétique
- Police : sans-serif système (cohérence avec l'hébreu).
- Style : **italique** pour signaler "transcription" et le distinguer de la traduction française (qui est en sérif droit).
- Taille : pilotée par le réglage hébreu (logique : c'est le même rôle, le texte primaire du psaume).

### Couleur
- Texte phonétique : `text.primary` (même hiérarchie visuelle que l'hébreu).
- Footer informatif : `text.secondary`.
- Aucune couleur d'accent ajoutée (l'italique suffit).

### Justification du choix sans-serif italique
- L'italique signale visuellement "ce n'est pas le texte original".
- Sans-serif évite la confusion avec la traduction française qui utilise New York (sérif).
- Lecture confortable pour des phrases avec diacritiques (é, ï, ô).

- [`Features/Psalm119/Psalm119SectionView.swift`](../ios/Tehilim/Features/Psalm119/Psalm119SectionView.swift) — idem.
- [`Features/Settings/SettingsView.swift`](../ios/Tehilim/Features/Settings/SettingsView.swift) — picker + footer + aperçu unifié.

### API du moteur

```swift
enum HebrewTransliterator {
    static func transliterate(_ source: String) -> String
}
```

Pure function, déterministe, sans état. Pas de cache nécessaire (très rapide, < 1 ms par verset typique).

### Performance
- Calcul à la volée dans `VerseRowView` via `phoneticText` propriété calculée.
- Mesure indicative : 176 versets du Tehilim 119 transcrits en < 50 ms total au défilement.
- Si problème de perf détecté : ajouter un cache mémoire `[Verse.id: String]` dans le repository.

### Persistance
Clé UserDefaults `pref.textMode`, valeurs `"hebrew"` (défaut) ou `"phonetic"`. Migration depuis V1.0 : aucune (clé absente → défaut hébreu, transparent pour l'utilisateur existant).
