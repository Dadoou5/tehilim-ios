## A2 — Tikkoun HaKlali

**Source** : tradition Rabbi Nachman de Breslev, *Likoutey Moharan*. 10 Tehilim dans l'ordre suivant : 16, 32, 41, 42, 59, 77, 90, 105, 137, 150.

**Implémentation** : entrée ajoutée dans `data/life_cases.json`, section "Spiritualité".

**À tester** : onglet Cas de la vie → section Spiritualité → tap "Tikkoun HaKlali" → la liste des 10 Tehilim s'affiche dans l'ordre.

## A3 — Partage de verset

**Geste** : long-press sur n'importe quel verset → menu contextuel iOS standard avec :
- **Copier** (texte plein avec attribution)
- **Partager** (image carrée 1080×1080 + ShareLink système)

**Carte de partage** (`VerseShareCard`) :
- 1080×1080 pour réseaux sociaux
- Tehilim N · numéro hébreu (titre)
- Texte hébreu (police large, RTL)
- Traduction française en italique (sous l'hébreu)
- Numéro de verset
- Attribution Beth Loubavitch + le-tehilim.online

**Conformité** : l'attribution Beth Loubavitch est imposée par l'autorisation reçue, présente sur chaque image partagée.

**Tests à valider device** :
- Long-press → menu apparaît.
- Tap "Copier" → presse-papier contient hébreu + français + attribution.
- Tap "Partager" → ShareLink natif iOS, image rendue, partage Messages/Mail/etc.

## A4 — Onboarding 3 écrans

**Conditions d'affichage** : `pref.onboarding.done == false`. Une seule fois par installation.

**Pages** :
1. **Bienvenue** — book icon — sources hébreu + traduction Beth Loubavitch.
2. **Au quotidien** — calendar icon — Tehilim du jour, cas de la vie.
3. **À ton rythme** — slider icon — réglages disponibles (taille, mode, traduction, rappel).

**Navigation** : TabView paginé (swipe), bouton "Suivant" / "Commencer", bouton "Passer" sur les 2 premières pages.

**Présentation** : `fullScreenCover` depuis `TehilimApp` → bloque tant que `onboardingCompleted == false`.

**Tests à valider device** :
- Première installation → Onboarding s'affiche au lancement.
- Tap "Passer" sur page 1 → ferme directement.
- Tap "Suivant" → page 2 → "Suivant" → page 3 → "Commencer" → ferme et marque l'état.
- Désinstaller / réinstaller → réapparait.

**Reset utilisateur** (debug) : effacer la clé UserDefaults `pref.onboarding.done`.

## A5 — Dédicaces personnelles

**Modèle** : `Dedication { id, hebrewName, latinName }`.

**Persistance** : JSON dans `Application Support/dedications.json`. Aucune donnée envoyée.

**UI** :
- Réglages → section "Dédicaces" → "Iluy Nishmat — gérer les dédicaces".
- Liste des dédicaces (vide par défaut).
- Bouton "Ajouter une dédicace" → sheet avec 2 champs (hébreu RTL + latin).
- Tap sur une entrée → édition.
- Swipe → suppression.

**Affichage** : `IluyNishmatBanner` lit la **première** dédicace de la liste. Si aucune, fallback sur la dédicace par défaut (Johann Meïr ben Sarah Bouganim — préservée pour les utilisateurs existants).

**Choix de design** :
- Liste plate, pas d'arborescence ni de catégories.
- Le footer explique que la première dédicace s'affiche en tête de psaume.
- Format suggéré dans les placeholders : "prénom hébreu + ben/bat + nom de la mère".

**Évolutions V1.5+** :
- Bascule cyclique entre plusieurs dédicaces actives (mode rotation).
- Yahrzeit : date hébraïque de décès + notification annuelle.
- Choix de la dédicace active dans l'UI (toggle / picker).

**Tests à valider device** :
- Avant ajout : `IluyNishmatBanner` affiche la dédicace par défaut.
- Ajout d'une nouvelle dédicace → `IluyNishmatBanner` la prend immédiatement (sans relance).
- Suppression de toutes les dédicaces → fallback sur la dédicace par défaut.
- Persistance : force-quit + relance → liste préservée.

## A1 — Widget verset du jour (différé V1.5)

**Pourquoi différé** : nécessite un **nouveau target Xcode** (Widget Extension), un **App Group** partagé entre l'app et le widget pour accéder aux données, et une `Timeline` qui calcule le Tehilim du jour à des intervalles donnés.

**Étapes prévues V1.5** :

1. Modifier `project.yml` pour ajouter un target `TehilimWidget` :
   ```yaml
   targets:
     TehilimWidget:
       type: app-extension.widgetkit-extension
       platform: iOS
       sources:
         - path: ios/TehilimWidget
       settings:
         INFOPLIST_FILE: ios/TehilimWidget/Info.plist
         PRODUCT_BUNDLE_IDENTIFIER: com.david.tehilim.widget
   ```

2. Créer un App Group `group.com.david.tehilim` (Apple Developer + entitlements).

3. Migrer les `UserDefaults` vers `UserDefaults(suiteName: "group.com.david.tehilim")`.

4. Implémenter `TimelineProvider` qui :
   - Lit `dailyMode` depuis le suite UserDefaults.
   - Charge `psalms.json` depuis le bundle (via `Bundle.main` du widget extension qui partage le bundle de l'app).
   - Calcule le Tehilim du jour via `DailyEngine`.
   - Retourne une entrée + une rotation horaire.

5. Designs widget : Small (1 verset court), Medium (verset + ref), Large (3-5 versets).

6. Configuration : taille small/medium/large par défaut, deep link `tehilim://daily` au tap.

**Effort** : 3-4h de dev pur + tests sur device. À faire sur un sprint dédié.
