> Document consolidé V1.5.

## Architecture

```
TehilimApp (target principal)
├── ios/Tehilim/...
└── embeds → TehilimWidget.appex (target widget)

TehilimWidget (target app-extension)
├── ios/TehilimWidget/
│   ├── Info.plist
│   ├── TehilimWidgetBundle.swift  (entry @main)
│   ├── DailyVerseWidget.swift     (configuration)
│   ├── DailyVerseProvider.swift   (TimelineProvider)
│   ├── DailyVerseWidgetView.swift (UI 3 tailles)
│   └── WidgetDataLoader.swift     (lecture JSONs)
└── sources partagées avec Tehilim :
    - ios/Tehilim/Core/Models/Psalm.swift
    - ios/Tehilim/Core/Models/DailyRules.swift
    - ios/Tehilim/Core/Services/PsalmRepository.swift
    - ios/Tehilim/Core/Services/DailyEngine.swift
    + data/*.json (resources embarqués)
```

## Comportement

- **Refresh** : à chaque minuit local (`Timeline.policy = .after(nextMidnight)`).
- **Mode** : monthly forcé en V1.5 (lit les règles du calendrier hébraïque, jour ↦ liste).
- **Contenu affiché** : 1er Tehilim de la liste du jour + son verset 1.
- **Tap** : `widgetURL("tehilim://daily")` → ouvre l'app sur l'onglet **Aujourd'hui**, pile vide.

### Tailles supportées

| Famille | Contenu |
|---------|---------|
| `systemSmall` | Tehilim N + numéro hébreu + 1er verset hébreu (4 lignes max) |
| `systemMedium` | Tehilim N + 1er verset hébreu (2 lignes) + 1er verset français |
| `systemLarge` | Liste complète du jour (chips) + Tehilim détaillé + verset HE+FR + indication tap |

## Limites V1.5

1. **Mode utilisateur ignoré** : si l'utilisateur a choisi le mode hebdomadaire, le widget reste en monthly.
   → V1.6 : configurer un App Group `group.com.david.tehilim` pour partager la pref.
2. **Aucune configuration utilisateur** : pas d'IntentConfiguration, pas de choix de mode dans l'éditeur du widget.
   → V1.6 : `AppIntentConfiguration` + paramétrage utilisateur.

## Test sur simulateur

1. Désinstalle l'ancienne version de l'app du simulateur (au cas où).
2. ⌘R dans Xcode (le widget se compile + s'embarque automatiquement).
3. Lance l'app une fois pour qu'elle soit présente.
4. ⇧⌘H pour aller à l'écran d'accueil du simulateur.
5. **Long-press** sur l'écran d'accueil → bouton **+** en haut à gauche.
6. Cherche "Tehilim" → 3 tailles proposées.
7. Choisis une taille → ajoute → place sur l'écran.
8. Le widget affiche le Tehilim du jour.
9. **Tape** dessus → l'app s'ouvre directement sur Aujourd'hui.

## Test sur device

Procédure identique. Sur device, le tap-to-launch est plus fiable qu'en simulateur. Bonus : le widget peut tourner même si l'app est désinstallée du multitâche.

## Refactoring associé

- `PsalmRepository.swift` est maintenant scindé : il ne contient plus que `PsalmRepository`. Les classes `LifeCaseRepository` et `Psalm119Repository` ont été déplacées dans `LifeCaseRepository.swift`. Cela permet au widget de ne dépendre que de `PsalmRepository.swift` sans avoir à embarquer les modèles `LifeCase` / `Psalm119Section` qu'il n'utilise pas.
