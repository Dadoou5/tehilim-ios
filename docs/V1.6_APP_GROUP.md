# V1.6 — App Group : préférences partagées entre app et widget

## Objectif

Le widget V1.5 lisait toujours en mode **mensuel**, ignorant le choix de l'utilisateur dans Réglages. V1.6 permet au widget de lire le `dailyMode` réellement configuré par l'utilisateur.

## Architecture

```
┌──────────────────────────┐         ┌──────────────────────────┐
│  Tehilim (app target)    │         │  TehilimWidget (extension)│
│                          │         │                          │
│  Preferences.dailyMode   │◄──┬───►│  DailyVerseProvider       │
│  Settings UI             │   │     │  (lit dailyMode)          │
└──────────────────────────┘   │     └──────────────────────────┘
                               │
                  UserDefaults │
                  (suiteName:  │
       "group.com.david.tehilim")
```

Les deux processus partagent **un seul `UserDefaults`** via App Group.

## Changements de code

### Nouveaux fichiers
- `ios/Tehilim/Core/Services/AppGroup.swift` — identifiant + accès UserDefaults partagé.
- `ios/Tehilim/Tehilim.entitlements` — entitlement App Groups.
- `ios/TehilimWidget/TehilimWidget.entitlements` — idem pour le widget.

### Modifiés
- `project.yml` :
  - `Tehilim` target : `CODE_SIGN_ENTITLEMENTS: ios/Tehilim/Tehilim.entitlements`
  - `TehilimWidget` target : `CODE_SIGN_ENTITLEMENTS: ios/TehilimWidget/TehilimWidget.entitlements`
  - `AppGroup.swift` ajouté aux sources partagées du widget.
- `Preferences.swift` : `dailyMode` est désormais sur `AppGroup.userDefaults`, non plus `.standard`.
- `TehilimApp.swift` : migration une fois au lancement — copie de `pref.dailyMode` depuis `.standard` vers le store partagé si besoin.
- `DailyVerseProvider.swift` : lit le mode depuis `AppGroup.userDefaults` au lieu de hardcoder `.monthly`.
- `SettingsView.swift` et `DailyView.swift` : `WidgetCenter.shared.reloadAllTimelines()` quand l'utilisateur change le mode → le widget se met à jour immédiatement.

## Test sur simulateur

1. ⌘R — installe app et widget.
2. Ajoute le widget à l'écran d'accueil simulateur.
3. Note le contenu (mode mensuel par défaut).
4. Ouvre l'app, va dans **Réglages → Lecture quotidienne → Mode → Jour de la semaine**.
5. Reviens à l'écran d'accueil — **le widget reflète le nouveau mode** (liste de psaumes du jour de la semaine).
6. Re-bascule sur **Cycle mensuel** dans Réglages → widget revient au mode mensuel.

Le `WidgetCenter.shared.reloadAllTimelines()` déclenche le refresh immédiat ; sinon le widget aurait attendu le minuit suivant.

## Préférences à terme migrer aussi

Pour V1.7+ si besoin d'autres données dans le widget :
- `pref.lastReadPsalmId` → utile pour un widget "Reprendre la lecture".
- `pref.translationFR` → si on veut que le widget montre/cache la traduction comme l'app.
- `favorites` (liste d'IDs) → utile pour un widget "Favoris" ultérieur.

Pour chaque pref migrée :
1. Changer son `@AppStorage` pour utiliser `store: AppGroup.userDefaults`.
2. Ajouter une migration dans `TehilimApp.migrateSharedPreferences()`.
3. Lire depuis le widget avec `AppGroup.userDefaults.X(forKey:)`.
