## Comment ça marche

### Traduction des Tehilim
- `Verse.translationFR` : traduction Beth Loubavitch (existait).
- `Verse.translationEN` : traduction JPS 1917 ajoutée en V1.7.
- Méthode `Verse.translation(for: TranslationLanguage)` choisit selon la pref.
- L'utilisateur change via **Réglages → Traduction → Langue**.

### UI Localization
Approche minimaliste : les chaînes françaises servent de **clés** dans `Localizable.strings`.

- `ios/Tehilim/Resources/fr.lproj/Localizable.strings` — clé = valeur en français.
- `ios/Tehilim/Resources/en.lproj/Localizable.strings` — clé en français → traduction anglaise.

SwiftUI `Text("Accueil")` → cherche "Accueil" dans le bundle de la locale active :
- iOS en français → trouve "Accueil" dans fr.lproj → affiche "Accueil"
- iOS en anglais → trouve "Accueil" dans en.lproj → affiche "Home"

Avantage : **aucun changement dans le code Swift**, les `Text("Accueil")` existants se traduisent automatiquement.

### Choix de la langue UI vs langue de traduction

**Important** : la langue UI (français/anglais) suit la langue système iOS. La langue de la traduction des Tehilim est **un choix séparé** dans Réglages.

Cas d'usage :
- iPhone en anglais + traduction Tehilim en anglais → tout en anglais. ✅
- iPhone en français + traduction Tehilim en anglais → UI FR, Tehilim avec traduction EN. ✅
- iPhone en anglais + traduction Tehilim en français → UI EN, Tehilim avec traduction FR. ✅
